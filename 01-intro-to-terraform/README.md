5. Запуск исправленного кода

```HCL
terraform {
  required_providers {
    docker = {
      source  = "kreuzwerker/docker"
    }
  }
  required_version = ">=1.12.0" /*Многострочный комментарий.
 Требуемая версия terraform */
}
provider "docker" {}

#однострочный комментарий

resource "random_password" "random_string" {
  length      = 16
  special     = false
  min_upper   = 1
  min_lower   = 1
  min_numeric = 1
}


resource "docker_image" "nginx"{
  name         = "nginx:latest"
  keep_locally = true
}

resource "docker_container" "nginx" {
  image = docker_image.nginx.image_id
  name  = "example_${random_password.random_string.result}"

  ports {
    internal = 80
    external = 9090
  }
}
```

6. Изменение имени контейнера

Заменяем

 - `name  = "example_${random_password.random_string.result}"`

на

  - name  = "hello_world"

![](img/img-01-10.png)

Ключ -auto-approve автоматически подтверждает применение изменений без показа плана выполнения и без запроса подтверждения. Это опасно, потому что нельзя увидеть, какие ресурсы будут созданы, какие изменены, какие удалены. А также не оставляет возможности ещё раз всё перепроверить, и если что-то не так, отменить операцию.

Может использоваться для CI/CD пайплайнов и скриптов автоматизации.
